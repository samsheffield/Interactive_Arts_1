For the first couple years you make stuff, it’s just not that good. It’s trying to be good, it has potential, but it’s not. But your taste, the thing that got you into the game, is still killer. CODE

A few things about code...

  • We give instructions to computers using very explicit rules that we call "code".
  • Code is programmed using programming languages (There are a lot of them!) (Not all are useful) (Almost all programming languages are rooted in English, with some exceptions.)
  • Programming languages are not communicative languages. We give specific commands following very strict rules already known to a computer. (Programming is quite bossy).
  • Instructions used to solve specific problems in code are called algorithms.
  • Bugs in code occur when commands are interpreted incorrectly.
  • Code is an interface.
  • Code is a material.
  • Code is a tool.

PROCESSING

"Processing is a programming language, development environment, and online community. Since 2001, Processing has promoted software literacy within the visual arts and visual literacy within technology. Initially created to serve as a software sketchbook and to teach computer programming fundamentals within a visual context, Processing evolved into a development tool for professionals. Today, there are tens of thousands of students, artists, designers, researchers, and hobbyists who use Processing for learning, prototyping, and production."

–from processing.org

BEFORE WE GET STARTED...

  • Processing is quite unique, powerful, and a great way to learn how to program (especially as an artist or designer!). But...
  • Processing is not a one-size-fits-all solution... It may not be able to scratch all of your code itches.
  • The good news! Your first language is the typically hardest to learn. Much of the concepts and conventions carry over from language to language
